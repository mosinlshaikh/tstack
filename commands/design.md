# /design

Review or create a product interface that is usable, accessible, responsive, and consistent with the product's brand.

## Evaluation areas
- User goals and task flow
- Information architecture
- Visual hierarchy and consistency
- Responsive behavior
- Accessibility and keyboard use
- Empty, loading, success, and error states
- Performance impact of visual effects
- Implementation feasibility

## Procedure
1. Identify primary users, devices, and critical tasks.
2. Map the shortest successful user journey.
3. Define reusable layout, spacing, type, and component rules.
4. Design all interaction states, not only the ideal screen.
5. Validate mobile-first behavior and content overflow.
6. Check contrast, focus, labels, motion, and touch targets.
7. Document implementation constraints and acceptance criteria.

## Guardrails
- Do not prioritize decoration over comprehension or speed.
- Avoid inaccessible motion, low contrast, and hidden controls.
- Do not claim visual validation without screenshots or a running interface.

## Output
- UX findings and recommended flow
- Component/state specification
- Responsive and accessibility requirements
- Implementation priorities
- Design acceptance checklist
