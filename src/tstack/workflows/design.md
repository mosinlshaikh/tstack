# TStack Design

## Objective
Produce usable, accessible, responsive and brand-consistent product experiences.

## Procedure
1. Define users, jobs, information hierarchy and critical journeys.
2. Review states, navigation, feedback, errors and recovery.
3. Validate responsive behavior, accessibility and performance constraints.
4. Translate decisions into implementation-ready specifications.

## Guardrails
- Visual novelty must not reduce usability.
- All critical actions need visible state and recovery.
- Accessibility failures on core journeys block approval.

## Output
Provide journey analysis, UI states, responsive rules, accessibility findings, specifications and acceptance criteria.
